# X-Tracker

X（旧Twitter）のアカウント関係管理・エンゲージメント最大化ツール。  
**手動操作のみ・規約完全準拠**で設計された個人用CRMです。

---

## 概要

「誰に・何をしたか・どう反応されたか」を記録し、効率的なエンゲージメント戦略を立てるためのツールです。  
自動投稿・自動フォロー等の機能は一切持ちません。記録・分析・判断をサポートし、実際のアクションは手動で行います。

---

## ディレクトリ方針（開発 / デプロイ分離）

- 実装・検証は `001_開発` で行う
- 配布・公開用の成果物は `../900_デプロイ` で管理する
- まず `001_開発` で動作確認し、問題ない内容だけを `900_デプロイ` に反映する

## 主な機能

### 1. ターゲット管理（`/targets`）
- アカウントを登録し、優先度・フェーズ・次回アプローチ日を管理
- フェーズ：`アプローチ候補 → 接触済み → 反応あり → 関係構築済み`
- 次回アプローチ日が来たカードはハイライト表示（⚡）
- `最近のやりとり相手を自動更新` ボタンで、X APIから候補を自動反映（新規追加 + 既存更新）
- 自動登録時にフォロワー数・フォロー数・FF比率・自分との規模差を見て `priority/tags/notes` を自動調整
- 自動登録時に、`ニッチキーワード` と相手プロフィールの一致度が低い候補は `クラスタ違い要チェック` タグを付与
- プロフィールの相互拡散系キーワードや極端なフォロー過多は `アルゴ汚染リスク要チェック` タグを付与
- 自分が相手をフォロー中かを `isFollowedByMe` で保存し、ターゲット一覧に表示
- 自動更新時に「自分のフォロワー」判定も行い、該当対象は `isFollowing=true / PARTNER` に更新して一覧から除外
- `FOLLOWED`（相手がフォロー）を記録した対象は自動で `PARTNER` に移行し、ターゲット一覧から除外
- `除外` 操作でアプリ内ブラックリストに追加し、提案・ターゲット一覧から除外（X上のブロック操作は行わない）

### 2. 引用チャンス（`/quote-opportunities`）
- `X API` の Recent Search で、自分のトピックに近い高会話投稿を検索
- 投稿本文・著者プロフィール・公開メトリクスから `引用チャンススコア` を算出
- 評価軸：トピック一致 / 返信・引用の伸びやすさ / 相手の規模 / 投稿鮮度 / ノイズリスク
- ニッチキーワード入力はブラウザに保存し、次回アクセス時に前回値を引き継ぎ
- 検索で使ったニッチキーワードをストックし、入力時に類似キーワードをサジェスト
- 検索結果は `QuoteCandidate` に保存し、同じ投稿の再検出回数・最新スコアを蓄積
- 候補投稿をXで開き、実際のリプライ・引用は手動で実行
- 有望な著者は `QUOTE_OPPORTUNITY` タグ付きでターゲット追加可能

### 3. インタラクション記録（`/interactions`）
- 投稿・リプライ・いいね・リポスト・引用・DM・フォロー・メンションを記録
- **結果**（反応なし / いいねされた / 返信された / フォローされた）を記録
- **反応の質**（ポジティブ / ニュートラル / ネガティブ）と**話題タグ**を記録
- 記録と同時に `Target.lastApproachedAt` を自動更新、`DailyActivity` を自動集計
- `最近のやりとりを一括同期` ボタンで、返信・メンション履歴から `Interaction` を自動生成

### 4. 行動量ログ（`/activity` / 補助機能）
- 1日の行動量を自動集計し、上限ゲージで可視化
- 上限80%到達で警告（⚠️）、100%到達で赤表示（🚫）
- 過去14日の履歴テーブル
- `活動量ログを更新` ボタンで、インタラクション履歴から過去14日を再集計（同期後のズレを補正）
- 現在は戦略導線を優先するため、サイドナビからは非表示

### 5. ダッシュボード（`/`）
- ターゲット数・総インタラクション数・高優先度数
- フェーズ別内訳・種類別インタラクション
- 戦略KPI：引用チャンス保存数 / 直近7日のターゲット追加数 / 直近90日の反応あり率
- 今日のアプローチ提案TOP5から、対象URLのコピー・記録・`除外` が可能

### 6. 自分の情報設定（`/settings`）
- フォローバックスコアの基準になる自分の情報を保存
- 保存項目：フォロワー数 / FF比率 / 平均エンゲージ率 / ニッチキーワード
- 提案スコア計算時に `myPersona` として利用
- `X API` 認証（Bearer Token）で自分情報を読み込み可能（`Xから読み込む`）

### 7. フォロー棚卸し（`/follow-audit`）
- `X API` から「自分がフォロー中」と「自分のフォロワー」を取得して差分を表示
- **自分はフォローしているが、相手はフォローしていない** アカウント（片思いフォロー）を一覧化
- フォロー稼ぎではなく、自分のフォロー運用整理（アルゴリズム負荷・ノイズ見直し）のために使う前提
- 一覧上で X プロフィールを直接開ける
- 既にターゲット登録済みの相手は `履歴を見る` からインタラクション確認が可能
- 実行時、X のフォロー一覧に含まれるターゲットへ `isFollowedByMe` を自動で `true` 更新

### 8. フォローメンテナンス（`/follow-maintenance`）
- ターゲットのうち **自分がフォロー中**（`isFollowedByMe`）の相手だけを一覧化
- **最終接触日が古い順**（未記録を優先）に並べ、あわせて **フォロワー規模が大きい** 順で二次ソート（引用・リプの波及を意識した候補出し）
- 画面内に、会話型リプ・引用リプライの意図（相手のフォロワーへの露出など）を短い説明文で表示
- `Xからフォロー中フラグを同期` でフォロー一覧 API とターゲットを突き合わせ（棚卸しと同様のフラグ更新・片思い一覧は棚卸し専用）
- 各 row から X プロフィール・インタラクション記録へ遷移。引用は `/quote-opportunities` へ導線

## データモデル

```
Target（アカウント）
  ├── priority:         HIGH | MEDIUM | LOW
  ├── phase:            PROSPECT | CONTACTED | ENGAGED | PARTNER
  ├── isFollowing:      相手が自分をフォローしているか（trueなら一覧から除外）
  ├── isFollowedByMe:   自分が相手をフォローしているか（一覧に表示）
  ├── lastApproachedAt: 最後のアプローチ日時（インタラクション記録時に自動更新）
  ├── nextApproachAt:   次回推奨アプローチ日（手動設定）
  ├── Interaction（インタラクション履歴）
  │     ├── type:       POST | REPLY | LIKE | REPOST | QUOTE | DM | FOLLOW | MENTION | OTHER
  │     ├── result:     NO_RESPONSE | LIKED | REPLIED | FOLLOWED | UNFOLLOWED
  │     ├── sentiment:  POSITIVE | NEUTRAL | NEGATIVE
  │     └── topic:      話題タグ（自由入力）
  └── Strategy（アプローチ戦略メモ）
        └── approach / result / nextAction

DailyActivity（日次行動量ログ）
  └── 日付ごとに replyCount / postCount / dmCount / followCount / likeCount / quoteCount を集計

QuoteCandidate（引用チャンス候補）
  ├── tweetId / conversationId / tweetUrl
  ├── authorUsername / authorFollowerCount
  ├── totalScore / recommendedAction
  ├── quoteScore / replyScore / repostScore / targetAddScore
  └── searchQuery / searchCount / lastSeenAt
```

## 行動量ガイドライン（X規約準拠の目安）

| アクション | 1日の目安上限 | 備考 |
|-----------|-------------|------|
| リプライ  | 100件       | 同一アカウントへの連続リプライは避ける |
| 投稿      | 50件        | 短時間の連投は避ける |
| DM        | 50件        | 非フォロワーへの送信は特に注意 |
| フォロー  | 400件       | フォロワー5,000人超は比率制限あり |
| いいね    | 1,000件     | 連続操作は避ける |
| 引用      | 50件        | |

※ X の公式上限は非公開。上記は運用上の経験則です。

## スコア仕様書（2026年根拠データ）

- フォローバックスコアの根拠データと判定基準: `docs/followback-score-spec-2026.md`
- 実装ライブラリ: `src/lib/score-rules-2026.ts`, `src/lib/score.ts`
- v0.3 追加要素:
  - `FF比率（Follower / Following）` を相互化しやすさ判定に利用
  - `ペルソナ類似度（0〜1）` を SimClusters 近似として加点
  - `calculatePersonaSimilarityByKeywords()` による簡易キーワード一致率（Jaccard）計算
- v0.4 追加要素:
  - `generateTargetingStrategy()` を追加（DBの直近エンゲージメントだけで「今日の狙い先」を生成）
  - 自分の規模に対する推奨ターゲット帯（0.3x〜3.0x）を見出しで提示
  - 小規模 / 中規模 / 高質反応の3セグメント別に日次アクションを提案
- v0.5 追加要素:
  - 自動更新時の要注意フラグ（`クラスタ違い要チェック` / `アルゴ汚染リスク要チェック`）を追加
- v0.6 追加要素:
  - `calculateQuoteOpportunityScore()` を追加
  - Recent Search の投稿・著者メトリクスから「引用 / リプライ / リポスト / 様子見」を提案
  - アルゴリズム倍率は公式確定値ではなく、運用仮説ベースの重みとして扱う
- v0.6.1 追加要素:
  - 引用チャンスを **アクション別スコア**（引用 / リプライ / リポスト / ターゲット追加）に分解
  - 直近90日の `REPLIED / FOLLOWED` 履歴から `PersonalHistoryStats` を生成し、スコアに過去成果ブーストを適用
  - xAIの `xai-org/x-algorithm` に倣い、`conversation_id` ベースの会話重複排除と著者多様性減衰を導入
  - UIで推奨アクションをハイライトし、過去成果サマリーを表示
- v0.6.2 追加要素:
  - `QuoteCandidate` テーブルを追加し、引用チャンス検索結果を保存
  - `tweetId` 単位でUPSERTし、再検索時は `lastSeenAt` と `searchCount` を更新
- v0.6.3 追加要素:
  - 引用チャンス画面のニッチキーワードをブラウザに保存し、次回アクセス時に自動復元
  - 最近使ったニッチキーワードを最大100件までチップ表示し、ワンクリックで再利用
  - 不要なキーワードチップを個別削除可能

## セットアップ

```bash
# 1. 依存関係
npm install

# 2. 環境変数
cp .env.example .env   # Windows: Copy-Item .env.example .env

# 2.1 X API を使う場合
# X_API_BEARER_TOKEN と必要なら X_MY_USERNAME を設定

# 3. DBマイグレーション
npm run db:migrate

# 4. 起動
npm run dev
```

→ http://localhost:3000 でアクセス

## 起動手順（CLI最小）

前提: 一度だけ `npm install` と `npm run db:migrate` が完了していること。

1. Cursorで `001_開発` フォルダを開く  
2. Cursorのターミナルで `npm run dev` を実行（起動はこの1コマンドだけ）  
3. ブラウザで `http://localhost:3000` を開く  
4. 連携は画面操作で実行  
   - `ターゲット` 画面でアカウント追加  
   - 対象行の `X同期` を押す  
   - 続けて `履歴取込` を押す  
5. `ダッシュボード` と `インタラクション` で反映確認

停止するときは、ターミナルで `Ctrl + C`。

## CLIを打たずに起動（Windows）

`001_開発/start-dev.bat` をダブルクリックすると、以下を自動実行します。

1. `.env` の存在確認（なければ `.env.example` から作成）
2. `node_modules` がなければ `npm install`
3. `npx prisma migrate deploy`
4. `npm run dev`

そのままブラウザで `http://localhost:3000` を開けば利用できます。

## X API認証について

- 現在実装している機能（`/api/x/user/*`, `/api/x/history/*`, `/api/x/me`, `/api/x/quote-opportunities`）は **Bearer Token だけで利用可能**
- これは「公開データの取得（read-only）」のみを扱っているため
- 将来、投稿・フォロー・いいね実行など「ユーザー操作」をAPIで行う場合は OAuth 2.0 User Context が必要
