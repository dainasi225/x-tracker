# Todo（優先度順）

このファイルは、製品化を見据えた「規約準拠・完全手動」の方針で実装を進めるための実装Todoです。  
上から順に進めれば、まずMVPが完成する構成にしています。

## 0. 開発方針（固定ルール）

- [ ] Xの操作はすべて手動（自動いいね・自動返信・自動フォローは実装しない）
- [ ] ユーザーに「安全上限」を提示し、上限超えを促すUI/文言を入れない
- [ ] 「フォロワー増加を保証する」表現は使わない（支援ツールとして表現）
- [ ] 主要な仕様変更時は `README.md` を更新する

## 1. MVP（最優先）

### 1-1. 投稿アイデア提案（毎日3案）
- [ ] ユーザーのBio/過去投稿からペルソナ要約を作成する
- [ ] 投稿案を3種類（失敗談 / 検証結果 / 共感質問）で生成する
- [ ] 各案に簡易スコアを付与する（理由付き）
- [ ] 「そのまま投稿せず、自分の言葉で調整」を促す注意文を表示する

### 1-2. 反応ユーザー収集
- [ ] 自分の投稿へのいいね・返信ユーザーを取得して一覧化する
- [ ] 既にフォロー済みのユーザーを除外する
- [ ] 同一ユーザーの重複を除去する
- [ ] 収集日時を表示する（データ鮮度の明示）

### 1-3. 優先ターゲット提案（3〜5人）
- [ ] 今日アクションすべき候補を3〜5人に絞って表示する
- [ ] 優先条件を実装する（同ニッチ / 規模近い / 会話履歴あり）
- [ ] 除外条件を実装する（大規模アカウント優先しない等）
- [ ] 各候補に「提案理由」を必ず表示する

### 1-4. 手動アクション導線
- [ ] 「Xで開く」リンクを実装する（実行はユーザー手動）
- [ ] 今日の安全上限（返信/いいね/フォロー）を表示する
- [ ] 当日の実行数を記録・表示する（上限管理のため）
- [ ] 上限到達時に注意表示を出す（ただし操作は強制しない）

### 1-5. 週次レポート（最小版）
- [ ] 週の純増フォロワー数を表示する
- [ ] 週の返信数 / フォローバック数を表示する
- [ ] 「今週効いた行動」を1〜2行で自動要約する
- [ ] 次週の提案を1つだけ表示する（やることを絞る）

## 2. 第2フェーズ（MVP後）

### 2-1. スコア説明の改善
- [ ] 予測スコアの内訳を可視化する（例: 規模一致 + 会話履歴）
- [ ] 説明文をユーザー向けのやさしい表現に統一する

### 2-2. 安全上限のカスタム
- [ ] 安全上限の初期値を提供する（返信3 / いいね5 / フォロー2）
- [ ] ユーザーが上限値を変更できる設定画面を作る
- [ ] 急な上げ幅に対する注意喚起を追加する

### 2-3. 投稿作成の時短
- [ ] 投稿テンプレ（冒頭フック / 本文 / 質問）を用意する
- [ ] 投稿案からテンプレに差し込む補助を実装する
- [ ] NG表現チェック（誇大・断定）を実装する

### 2-4. 学習ループ
- [ ] 行動ごとの結果を蓄積する（返信起点など）
- [ ] 成功パターン/失速パターンを自動抽出する
- [ ] 翌週提案の精度を改善する

## 3. 後回し（初期は実装しない）

- [ ] 高度な機械学習モデル（初期はルールベースで運用）
- [ ] 複雑な分析ダッシュボード（まずは行動導線を優先）
- [ ] 複数SNS同時対応（まずXで勝ち筋を作る）
- [ ] 自動実行に見える機能（規約リスク回避のため）

## 4. 受け入れ基準（MVP完了条件）

- [ ] 毎日15分以内で「投稿→反応確認→3〜5人提案→手動実行」が回せる
- [ ] 手動以外でXアクションが実行されない
- [ ] 週次レポートで最低限の成果確認ができる
- [ ] READMEに最新の運用手順と制約が反映されている

## 5. 運用メモ

- [ ] 仕様変更時はこのTodoの優先度も必ず見直す
- [ ] 迷ったら「規約準拠」「手動優先」「行動を絞る」を優先する
