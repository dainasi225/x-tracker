# フォローバックスコア仕様書（2026年 X 実データベース）

この文書は、フォローバック予測スコアの「根拠データベース」と「判定基準」を定義する。

| 要素 | 加点/減点 | 根拠（2026年） |
|---|---:|---|
| 返信（直近7日） | +40 | Replies は Likes より圧倒的に強いシグナル。会話発生で確率上昇 |

## 詳細根拠（2026年版・優先反映）

### 返信（直近7日） +40（実装では +42 チューニング）
- RepliesはLikesの13.5〜27倍の重みレンジが観測される。
- 会話成立（返信往復）で75〜150倍級の強いシグナルになりやすい。
- 返信された対象のフォローバック率は、Likes中心の対象より3〜8倍高い傾向。
- 最新性重視のため、直近7日を最優先で評価する。

### いいね（直近7日） +18
- Likesは基準値1xまたは0.5x相当の弱いシグナルとして扱う。
- 単発Likesのみではリーチ拡大・フォローバック寄与が小さい。
- 補助シグナルとして維持し、返信や高質反応と組み合わせて評価する。

### 高質反応（positiveScore） +7〜+14
- Quote TweetはLikes比で約25x、Repostは約20xの強シグナル。
- 詳細コメント、返信＋いいね等の複合反応は「深い関与」を示す。
- 高いポジティブ反応はフォローバック率を10〜20%押し上げる傾向。

### フォロワー帯（小規模優位）
- `<3,000`: +16（最優先ターゲット帯）
- `<5,000`: +14（実装で新設）
- `<10,000`: +8（実装で新設）
- 根拠: 小規模層は相互率・エンゲージメント率が高い。

### フォロワー帯（大規模ペナルティ）
- `>50,000`: -18
- `>80,000`: -25
- 根拠: 大規模になるほどフォローバック難易度が上がる。

### 累積接触 >= 5回 +13
- 複数回関与は関係性の深さとして認識される。
- 単発接触よりフォローバック率が上がりやすい。

### 14日以上接触なし 最大 -25
- 時間減衰が大きく、最新性を失うほど価値が下がる。
- 14日超は優先順位を下げ、再活性化施策を検討する。

## 2026年の運用方針（優先）

- 返信を最優先に設計する（Likes偏重を避ける）。
- 小規模層（特に3,000未満）を成長の主ターゲットにする。
- 直近7日を最重要期間として扱い、14日超は減衰を強く見る。

## v0.3 追加仕様（優先）

### 返信の会話成立を分離
- 単発返信（7日以内）: +40
- 会話成立（7日以内）: +58
- 単発返信（14日以内）: +22
- 会話成立（14日以内）: +32

### 相対フォロワー比（自分基準）
- 相対比 `0.3〜3.0x`: +12（黄金ゾーン）
- 相対比 `>10x`: -15
- 相対比 `<0.1x`: +6

### データ項目（実装）
- `myFollowerCount`: 自分のフォロワー数
- `hasAuthorReplyBack`: 自分の返信後に相手返信があったか
- `conversationDepth`: 会話深さ（返信関連の往復数の簡易指標）

## v0.4 追加仕様（FF比率 / ペルソナ類似）

### FF比率（Follower / Following）
- 指標: `followerCount / followingCount`
- ルール: `1.5未満`で +8
- 目的: フォロー返しやすさを絶対フォロワー数以外でも評価する

### ペルソナ類似度（SimClusters近似）
- 指標: `personaSimilarity`（0〜1）
- 加点: `round(personaSimilarity * 18)`（最大+18）
- 目安: `0.7以上`は高類似として優先提案

### 追加データ項目（実装）
- `followingCount`: 相手のフォロー数
- `personaSimilarity`: ニッチ/興味の一致度（手動入力から開始可能）
- `UserPersona`: 将来の自動類似度計算用インターフェース

## 判定ラベル（%表示の目安）

- 75以上: かなり有望
- 60〜74: おすすめ
- 40〜59: 関わると効果ありそう
- 40未満: 今は様子見
